Phase 1: Defining the .ay Header
Before you compress any pixels, your program needs to know how to read your file. You are essentially creating a custom struct that will sit at the very top of the binary data.
The Magic Number: The first two bytes of your file should be the characters 'A' and 'Y'. When your decompressor runs, it checks these first. If they aren't there, the program throws an error: "Invalid file format!"
Metadata: Immediately after the magic number, write out the image dimensions. You will need to save the width (4 bytes/int), height (4 bytes/int), and channels (e.g., 3 for RGB).

Phase 2: Run-Length Encoding (RLE)
RLE is brilliant for images with large areas of flat color (like backgrounds or simple graphics).
The Logic: Convert your 2D image into a flat 1D array of pixels. Loop through the array. Keep a counter of how many identical pixels you see in a row.
The Output: Instead of saving {Red, Red, Red, Red, Blue}, you save {4, Red, 1, Blue}.
The Catch: What if a run is longer than 255 pixels? A standard byte can only hold up to 255. You'll need to write logic that says, "If the count hits 255, save the chunk and start a new count."

Phase 3: Huffman Coding (The Boss Level)
Once RLE squashes the continuous runs, Huffman Coding squashes the data based on frequency.

Frequency Map: Count how often every unique color (or unique RLE sequence) appears.

The Binary Tree: Build a priority queue (min-heap) to construct your Huffman tree. Assign the most common elements short binary codes (like 0 or 10) and rare elements long codes (like 11010).

The Dictionary Table: You must save your Huffman tree (or the frequency table) into the header of your .ay file! If you don't save the dictionary, the decompressor will have no idea how to translate the binary codes back into colors.

Phase 4: Bitwise I/O in C++
This is where the real engineering happens. Standard C++ streams (std::ofstream) write data in 8-bit bytes. If your Huffman code is 101 (3 bits), you can't just write that to a file directly.

The Bit Buffer: You will need to create an unsigned char (a 1-byte buffer) and an integer keeping track of how many bits are "filled."

Bit Shifting: Use the bitwise OR | and shift << operators to pack your variable-length Huffman codes into your 8-bit buffer. Once the buffer hits 8 bits, you write it to the .ay file and reset it.

Phase 5: The Decompressor
To prove your project works from start to finish, your main() function should essentially look like this:

Read input.png.

Compress and save as output.ay.

Open output.ay and read the header.

Decompress the binary payload back into a pixel array.

Save the result as output.png.
(If your code is flawless, output.png will be identical to input.png, but output.ay will have a much smaller file size).
